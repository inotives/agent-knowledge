# EP-00003 — v0.3 Security, Maintenance & LLM Review

## Problem / Pain Points
- No protection against agents accidentally logging API keys, tokens, or credentials in turns or pages
- No maintenance tools for system health — no way to check for orphaned pages, stale content, or purge old sessions
- MCP prompts not implemented — agents have no playbook for session bootstrap/wrapup workflows
- `akw review` (LLM-powered daily review) not yet built — the automated bridge from session drafts to knowledge drafts
- CLI missing `akw purge` and `akw reindex` commands

## Suggested Solution
Add security safeguards, operational maintenance tools, MCP prompts for agent guidance, and the LLM-powered CLI review command. This makes the system production-ready.

## Implementation Phases

### Phase 1 — Secret Scanning
- [ ] Create `src/agent_knowledge/core/sanitizer.py` — scan content for common secret patterns (API keys, tokens, connection strings, private keys, passwords)
- [ ] Integrate into `session_log`, `memory_create`, `memory_update` — redact or reject content containing secrets
- [ ] Tests for secret pattern detection and redaction

### Phase 2 — Maintenance Tools
- [ ] `maintain_get_stats` MCP tool — orphaned pages (no inbound links), stale pages (no updates in N days), page counts per tier
- [ ] `maintain_purge` MCP tool — delete reviewed sessions + turns older than retention period, clean up associated draft files
- [ ] `akw purge [--older-than 365]` CLI command
- [ ] `akw reindex` CLI command
- [ ] Tests for maintenance tools

### Phase 3 — MCP Prompts
- [ ] `session-bootstrap` prompt — instructs agent to start session, check pending reviews, load context
- [ ] `session-wrapup` prompt — instructs agent to summarize turns, write session draft, end session
- [ ] Register prompts via FastMCP

### Phase 4 — LLM-Powered CLI Review
- [ ] Decide LLM SDK: `anthropic` SDK (simple, matches default config) vs `litellm` (multi-provider)
- [ ] Add LLM SDK as optional dependency
- [ ] `akw review` — fetch pending session drafts, call LLM to synthesize knowledge drafts, write to `/memory/drafts/knowledge/` and review report to `/memory/drafts/reviews/YYYY-MM-DD.md`, call `review_complete`
- [ ] Tests for review command (mock LLM calls)
