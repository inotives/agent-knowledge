# EP-00002 — v0.2 Review Pipeline & Promotion

## Problem / Pain Points
- v0.1 stores sessions and turns but they never become knowledge — the pipeline stops at raw data
- No review tools exist to synthesize session drafts into knowledge drafts
- No promotion tools to move drafts into curated knowledge or skills
- `session_start` context loading is stubbed — agents don't receive relevant skills/knowledge on startup
- No CLI inspection or review commands for users

## Suggested Solution
Complete the knowledge maturation pipeline: session review → daily review → promotion → context loading. Add CLI commands for review and inspection. This makes the system end-to-end functional.

## Implementation Phases

### Phase 1 — Review Tools
- [x] `review_get_pending` MCP tool — returns orphaned sessions with raw turns + unreviewed session drafts
- [x] `review_complete` MCP tool — sets `reviewed_at` on sessions, deletes session draft files
- [x] Storage: `get_unreviewed_sessions`, `get_sessions_needing_drafts`
- [x] Wire orphan auto-close in `session_start` (verified)
- [x] Tests for review tools (8 new tests)

### Phase 2 — Promotion Tools
- [x] `promote_to_knowledge` MCP tool — moves from drafts/knowledge/ to knowledge/, records edits, updates search index
- [x] `promote_to_skill` MCP tool — moves from knowledge/ to skills/, records edits, updates search index
- [x] Path validation (source/target tier enforcement)
- [x] Tests for promotion

### Phase 3 — Session Context Loading
- [x] `session_start` recommended context returns actual page content inline (not just paths)
- [x] Matches skills by project tags, includes recent knowledge pages

### Phase 4 — CLI Review & Inspection
- [x] `akw sessions [--project X] [--date Y]` — list sessions with turn counts and last request
- [x] `akw search "query" [--tier T]` — search memory from terminal with sync before search
- [ ] `akw review` — deferred to EP-00003 (requires LLM SDK dependency decision)

## Out of Scope (EP-00003+)
- `akw review` (LLM-powered daily review CLI — requires SDK dependency decision)
- MCP prompts (`session-bootstrap`, `session-wrapup`)
- Secret scanning (write-time content sanitization)
- `maintain_get_stats`, `maintain_purge`
- CLI `akw purge`, `akw reindex`
